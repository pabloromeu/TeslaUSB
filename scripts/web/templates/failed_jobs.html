Failed Jobs

Tasks that hit their retry cap. Review the error, fix the cause, then retry.

What does this mean?

A failed job is a background task — archive copy, video index, cloud upload, or live event upload — that retried multiple times and gave up. Each card shows what failed and a recommendation for what to do.

Failed vs queued: a failed job has stopped retrying and needs your decision. A "queued" or "pending" item on the Health card is just waiting in line for the worker — it isn't failed, it'll process when the worker reaches it. Big queue numbers (hundreds queued) mean the worker is catching up after a backlog, not that anything is broken.

Retry resets the row so the worker picks it up again. Use this when the underlying cause is fixed (network is back, free space is restored, permissions corrected).

Delete removes the queue row only — it never deletes the underlying video file from disk. Producers (file watcher, boot scan, archive worker) may legitimately re-enqueue the same file later if it still exists.

Clip value badge: Event clip = Tesla recorded it because something happened (high value). Rolling buffer = RecentClips, written continuously while the car is powered (driving or Sentry standby) and rotated hourly. Map / trip data = re-derivable from the video.
